## はじめに

「外出中にバグ報告が来た」「移動中にコードレビューを終わらせたい」――そんな場面で、スマホやタブレットから GitHub Copilot の力を借りられると助かります。

PC なしでも GitHub Copilot が活躍できる場面は意外と多く、うまく組み合わせることで外出先でも開発の流れを止めずに済みます。この記事では、モバイル環境での具体的な活用方法を紹介します。

---

## スマホから使える GitHub Copilot の入口

### 1. GitHub Mobile アプリ（iOS / Android）

GitHub の公式モバイルアプリには **Copilot Chat** が統合されています。

**できること：**

- リポジトリのコードを読みながら Copilot に質問する
- Issue・PR の内容を要約してもらう
- バグの原因を特定するヒントをもらう
- コードの変更点を解説してもらう

**使い方：**
アプリ下部のナビゲーションから「Copilot」アイコンをタップするとチャット画面が開きます。コードを貼り付けたり、Issue の URL を共有したりしながら会話できます。

```
# 外出先での使い方の例
「このエラーログを見てください。原因と対処法を教えて」
[エラーメッセージをペースト]
```

---

### 2. GitHub.com（モバイルブラウザ）

スマホのブラウザから `github.com` にアクセスしても Copilot Chat が使えます。

PC と同じ機能が使えるため、アプリより操作の自由度が高い場面もあります。特にコードを選択して Copilot に質問する操作は、ブラウザ版が使いやすいです。

**ポイント：**
- Safari / Chrome でデスクトップ版サイトを表示するとレイアウトが崩れにくい
- iOS は Safari、Android は Chrome が安定している

## 外出中にできる主な作業

### コードレビュー

PR の差分を確認しながら Copilot に解説してもらえます。

```
# GitHub Mobile で PR を開いた状態で
「このPRの変更点を要約して。気になる点があれば指摘してください」
```

レビューのコメント下書きを Copilot に手伝ってもらい、帰宅後にそのまま投稿するという使い方が効率的です。

---

### Issue のトリアージと返信

バグ報告や質問 Issue が来たとき、Copilot に内容を理解させて返信文を生成させることができます。

```
# Issue の内容を Copilot Chat に貼って
「この Issue に対して、原因の可能性と確認してほしいことを丁寧に返信する文章を書いてください」
```

外出先でも Issue を「放置」せず、素早く一次対応できます。

---

### ドキュメント・README の修正

文章の修正や追記は、スマホでも十分できる作業です。

```
# README の一部を貼り付けて
「この説明をより分かりやすく書き直してください。対象読者は初学者です」
```

---

### エラー調査・原因特定

Slack やメールで届いたエラーログを GitHub Mobile の Copilot Chat に貼ることで、原因のあたりをつけられます。

```
「次のスタックトレースを見てください。どのファイルのどの処理で失敗していますか？」

Error: ECONNREFUSED 127.0.0.1:5432
    at TCPConnectWrap.afterConnect [as oncomplete] (node:net:1187:16)
```

PC に戻ったときに何をすべきかが明確になり、帰宅後の作業がスムーズになります。

## タブレット（iPad）があるとさらに広がる

iPad を持ち歩いている場合、選択肢が一気に増えます。

### GitHub Codespaces をブラウザで使う

Safari または Chrome から `github.dev` または Codespaces を開くと、**フルの VS Code 環境**がブラウザで動きます。Copilot の補完も使えます。

```
# アクセス方法
github.com のリポジトリページで「.」キーを押す
→ github.dev（軽量エディタ）が開く

# Codespaces を使う場合
「Code」→「Codespaces」→「Create codespace」
```

外付けキーボードと組み合わせれば、簡単なバグ修正や設定ファイルの変更なら iPad だけで完結できます。

### Working Copy（iOS Git クライアント）

iOS 向けの Git クライアントアプリで、ローカルでファイルを編集してコミット・プッシュができます。`github.dev` と組み合わせると作業の幅が広がります。

## 外出先での Copilot 活用フロー（実例）

### 移動中にバグ報告を受け取った場合

```
1. GitHub Mobile でエラーログを確認
2. Copilot Chat に「原因は何か、どのファイルを見ればよいか」を聞く
3. 関連するファイルを開いてコードを確認
4. Copilot に修正方針を聞いて、メモアプリに書き留める
5. 帰宅後、メモをもとに実装して修正
```

### レビュー待ちの PR がある場合

```
1. GitHub Mobile で PR の差分を確認
2. Copilot Chat に「この変更の影響範囲と潜在的な問題点を教えて」と聞く
3. コメントの下書きをメモアプリに書き留める
4. 帰宅後、下書きをもとにレビューコメントを投稿
```

## スマホ操作を快適にするヒント

### テキスト展開アプリを使う

よく使う質問テンプレートをテキスト展開アプリ（iOS: Textexpander / Espanso など）に登録しておくと入力が楽になります。

```
# 例: "//review" と入力すると展開される
このコードのレビューをお願いします。
特に確認してほしい点：
- セキュリティの問題がないか
- エラーハンドリングが適切か
- パフォーマンス上の懸念がないか
```

### 音声入力を活用する

長い質問文を打つのが面倒な場合、音声入力（iOS の dictation 機能）が便利です。公共の場では使えませんが、移動中の電車内以外では十分活用できます。

## モバイルでできないこと・向かないこと

正直に書いておきます。

| 作業 | 向き不向き |
|------|-----------|
| コードレビュー（読むだけ） | ◎ 向いている |
| Issue トリアージ・返信 | ◎ 向いている |
| ドキュメント修正 | ○ できる |
| バグの原因調査 | ○ できる |
| コード修正・実装 | △ iPad + キーボードなら可 |
| テスト実行・デバッグ | △ Codespaces があれば可 |
| 複雑なリファクタリング | ✕ 向いていない |
| ローカル環境が必要な作業 | ✕ 不可 |

スマホでの作業は「準備」や「調査」に集中して、「実装」は PC に任せるのが現実的です。

---

## まとめ

GitHub Copilot のモバイル活用をまとめます。

- **GitHub Mobile アプリ**：Copilot Chat でコード質問・Issue 対応・PR レビュー補助
- **モバイルブラウザ**：github.com の Copilot Chat をそのまま使える
- **iPad + Codespaces**：軽微な修正ならフルの開発環境として使える

「外出中は開発を諦める」から「外出中でも少し前に進める」へ。Copilot をうまく使えば、スキマ時間の生産性が変わります。
